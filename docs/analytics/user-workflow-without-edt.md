# Рабочий день с CDT 41 без EDT (чеклист пользователя)

Цель: зафиксировать **практический** сценарий использования расширения рядом с Git и внешними инструментами 1С. Детали команд — в [README.md](../../README.md).

## Утро: открыть конфигурацию

1. Клонировать репозиторий с выгрузкой **Designer** или дерева **EDT**.
2. Открыть корень конфигурации в VS Code / Cursor (`Файл → Открыть папку`).
3. Убедиться, что панель **CDT 41** видна в Explorer; при необходимости: **CDT 41: Open Metadata Tree** или `Ctrl+Alt+1` / `Cmd+Alt+1`.

## Навигация

1. Разворачивать узлы по мере необходимости (ленивая загрузка).
2. **Поиск:** `Ctrl+F` / `Cmd+F` в фокусе дерева; при необходимости включить поиск по синониму/комментарию или regex.
3. **Фильтр по типу** или **по подсистеме** — через команды палитры или контекстное меню подсистемы.

## Правка метаданных

1. Выбрать объект → **Show Properties** для панели свойств **или** **Open XML File** для прямого редактирования.
2. Операции **Create / Duplicate / Delete / Rename** — из контекстного меню (где поддерживается типом объекта).
3. После внешнего изменения XML на диске дерево должно обновиться через watcher; при сомнениях — **Refresh** или **Clear tree cache**.

## Формы и роли

1. **Форма:** для `Ext/Form.xml` — открыть **Редактор форм** (команда из дерева для узла формы). Учитывать статус прототипа в [README.md](../../README.md).
2. **Роль:** узел роли → **Open Rights Editor**; сохранение — `Ctrl+S` в webview редактора прав (см. README).

## BSL

1. Общие модули и файлы `.bsl` открывать через дерево или обычный Explorer.
2. Подсветка и навигация по коду — через **BSL Language Server** и другие расширения; CDT не дублирует LSP.

## Сообщить о проблеме

1. **Output → CDT 41:** при необходимости приложите фрагмент лога (как в шаблоне issue).
2. **CDT 41: Copy diagnostics summary** (палитра команд): вставьте текст в issue — там версия расширения, VS Code, папки workspace и найденные корни конфигурации (Designer/EDT). Удобно для багрепортов и поддержки. Расшифровка каждой строки: [diagnostics-summary-field-dictionary.md](./diagnostics-summary-field-dictionary.md).
