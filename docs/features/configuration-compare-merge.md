# Сравнение и объединение конфигураций

MVP compare-view открывается командой **CDT 41: Сравнить/объединить конфигурацию из файлов...** на узле корня конфигурации. Левая сторона берётся из текущего workspace, правая выбирается как каталог файловой выгрузки.

## Что сравнивается

- Метаданные индексируются из XML и сопоставляются по uuid, затем по квалифицированному имени.
- Конфликты идентичности подсвечиваются отдельно: одинаковое имя при разном uuid или одинаковый uuid при разном имени требуют ручного решения.
- BSL-модули сравниваются не построчно, а по процедурам и функциям. Добавленная процедура считается отдельным изменением; сдвиг соседнего кода сам по себе не создаёт конфликт.
- Модули целиком `только слева` / `только справа` отображаются как различия, чтобы они не терялись в статистике.

## Автоматическое объединение BSL

Нижний слой merge поддерживает безопасный logical merge для процедур: он принимает только заранее построенный approved preview, повторно проверяет guard перед записью, делает backup и блокирует сомнительные случаи. Если меняется существующий блок внутри процедуры, решение остаётся за пользователем.

Командный слой compare-view пока только открывает сравнение и не пишет файлы сам. Это сделано намеренно: UI выбора, approve и execute должны использовать тот же preflight/executor контракт, а не обходить его.

## Ограничения MVP

- Структурное объединение XML-метаданных пока read-only.
- Полноценный 3-way merge конфигураций ещё не реализован.
- Интерактивный выбор источника правды для metadata identity conflict будет отдельным шагом.
