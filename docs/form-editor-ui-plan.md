# План улучшения UI/UX редактора форм 1С

Редактор форм — webview в расширении 1cviewer для просмотра и редактирования структуры формы из `Ext/Form.xml`. Документ фиксирует текущее состояние визуальной части, выявленные проблемы и конкретные предложения по улучшению без изменения логики и кода (только план).

---

## 1. Текущее состояние

### 1.1 Структура webview

- **Верхняя строка (top-row):**
  - **Слева:** зона дерева элементов (`.zone-tree`), ширина по умолчанию 280px, изменяется сплиттером.
  - **Справа:** зона свойств (`.zone-props`), занимает оставшееся место.
- **Горизонтальный сплиттер** (6px) между деревом и свойствами.
- **Нижняя зона:** превью (`.zone-preview`), высота по умолчанию 200px, изменяется вторым сплиттером. Внутри — вкладки «Форма» и «Модуль».

### 1.2 Используемые технологии и стили

- Один HTML-документ со встроенными `<style>` и `<script>` в `formEditorProvider.ts` (метод `getWebviewHtml`).
- Шрифты и цвета завязаны на переменные VS Code: `--vscode-font-family`, `--vscode-font-size`, `--vscode-foreground`, `--vscode-editor-background`, `--vscode-panel-border`, `--vscode-list-hoverBackground`, `--vscode-list-activeSelectionBackground`, `--vscode-input-*`, `--vscode-button-*` и др.
- Типографика: заголовки зон — `h3`, размер `0.9em`, цвет `--vscode-descriptionForeground`. Основной текст — стандартный размер и цвет.
- Отступы: в зонах в основном `padding: 8px`; у узлов дерева `2px 4px`; у превью-элементов `4px 8px`.
- Кнопки: скругление `border-radius: 3px`; у узлов дерева и части блоков — `2px`.
- Сплиттеры: фон `--vscode-panel-border`, при hover — `--vscode-focusBorder`.

### 1.3 Компоненты

| Компонент | Описание |
|-----------|----------|
| Дерево элементов | Список узлов вида «Имя (Тег)», поддержка drag-and-drop, выделение, классы `.tree-node`, `.tree-children`, для таблиц — `.tree-table-columns`. |
| Панель свойств | Заголовок выбранного элемента, список полей: тип (только текст), имя, ID, остальные свойства из модели, секция «События» с полем и кнопкой «Перейти». Кнопки «Отмена» и «Сохранить» внизу. |
| Превью | Вкладки «Форма» / «Модуль». Форма: плейсхолдер «Превью не реализовано» или дерево контролов с drag-and-drop. Модуль: кнопка «Модуль формы» и подсказка. |
| Empty state | Текст + подсказка при отсутствии/ошибке формы. |
| Ошибки | Красный текст (`.error`) с сообщением. |

## 2. Выявленные проблемы

### 2.1 Общий вид и согласованность с VS Code

- Левая панель использует `--vscode-editor-background`; в VS Code боковые панели обычно используют `--vscode-sideBar-background` — дерево визуально не отделено от области редактора.
- Смешение скруглений: 2px (узлы, инпуты превью, таблицы) и 3px (кнопки) — нет единой шкалы.
- Вкладки «Форма» / «Модуль» оформлены как обычные кнопки (primary), тогда как в VS Code вкладки обычно выглядят иначе (подчёркивание активной, нейтральный фон).

### 2.2 Дерево элементов

- Все узлы развёрнуты всегда; нет сворачивания/разворачивания веток — при большой форме навигация затруднена.
- Нет иконок по типу элемента (Button, InputField, Group и т.д.) — сложнее быстро сканировать дерево.
- Визуально не различаются контейнеры (Group, Page, Table) и конечные элементы.
- Отступ вложенности фиксированный (12px), уровень вложенности при большом числе уровней слабо читается.
- Текст узла «Имя (Тег)» при длинных именах и тегах может обрезаться или переноситься без явного правила (нет `overflow`/`text-overflow`).

### 2.3 Панель свойств

- Все свойства в одном плоском списке; «События» выделены только жирным заголовком — нет группировки в блоки (Основные, Внешний вид, События и т.д.).
- Фиксированные ширина подписи (100px) и поля (200px): на узкой панели свойств возможны наложения или неудобная прокрутка; нет адаптивности.
- Заголовок выбранного элемента (имя + тег) и кнопки «Отмена»/«Сохранить» визуально не отделены от списка свойств — иерархия слабая.
- Нет разделителей между группами полей; тип элемента показан обычным текстом — можно оформить как badge (капсула).

### 2.4 Превью

- Текст «Превью не реализовано» создаёт ощущение незавершённости; при сохранении этого состояния лучше оформить как явный empty state с коротким пояснением.
- Вкладки выглядят как две кнопки действия, а не переключатель контекста.
- В превью формы элементы имеют границы и отступы; визуально не ясно, что это «макет», а не интерактивная форма (можно добавить лёгкий фон или подпись).

### 2.5 Сплиттеры и отступы

- Сплиттер — тонкая полоска 6px без «ручки»; неочевидно, что её можно тянуть (особенно для новых пользователей).
- Отступы заданы разрозненно (2, 4, 6, 8, 12, 14, 16px); нет единой шкалы (например, 4 / 8 / 12 / 16).

### 2.6 Читаемость и контраст

- Заголовки зон (h3) в `descriptionForeground` могут быть слишком приглушёнными в некоторых темах.
- Плейсхолдер «Выберите элемент» и пустые состояния — только курсив и серый цвет; можно усилить за счёт размера или иконки.

### 2.7 Кнопки и действия

- Кнопки «Отмена» и «Сохранить» внизу панели свойств при длинном списке свойств уезжают вниз — не закреплены (sticky) и не всегда на виду.
- Статус «Сохранено» / «Сохранение...» выводится текстом рядом с кнопками — можно оформить компактнее (иконка + краткий текст) или вынести в общую область уведомлений.

### 2.8 Доступность и фокус

- Видимость фокуса есть только у `input` (outline); у кнопок и узлов дерева явных стилей фокуса при навигации с клавиатуры не описано.
- Нет ARIA-атрибутов для дерева и областей (роли, aria-expanded, aria-selected и т.д.).

## 3. Цели улучшений

- Визуальная и поведенческая согласованность с VS Code (темы, панели, вкладки, фокус).
- Улучшение читаемости и иерархии: типографика, группировка, контраст.
- Удобство навигации: дерево с возможностью сворачивания, иконки по типу элемента.
- Предсказуемая сетка отступов и скруглений.
- Более понятный UX панели свойств (группы, адаптивность, закреплённые действия).
- Улучшение восприятия сплиттеров и пустых состояний без изменения логики приложения.

## 4. Предложения по разделам

### 4.1 Общий вид и темы

- Задать для `.zone-tree` фон `var(--vscode-sideBar-background)` (и при необходимости цвет текста от sidebar), чтобы левая панель визуально соответствовала боковой панели VS Code.
- Ввести единую шкалу скруглений, например: 2px — мелкие элементы (узлы, чипы), 4px — инпуты и карточки, 6px — кнопки (или придерживаться 3px, если хотим ближе к текущим кнопкам).
- Вынести повторяющиеся значения отступов в CSS-переменные (например `--fe-spacing-xs: 4px; --fe-spacing-sm: 8px; --fe-spacing-md: 12px; --fe-spacing-lg: 16px`) и применять их в зонах, списках и формах.

### 4.2 Дерево элементов

- Добавить сворачивание/разворачивание веток: иконка-шеврон (или треугольник) слева от контейнеров, по клику переключать видимость дочерних узлов; сохранять состояние развёрнутости в данных или в `data-*` и при перерисовке восстанавливать.
- Добавить иконки по тегу элемента (например, иконка кнопки для Button, поля ввода для InputField, папки для Group/Page) — можно использовать Unicode-символы или inline-SVG из набора VS Code (codicon), если разрешить загрузку/вставку ресурсов в webview.
- Визуально отличать контейнеры от листьев: например, чуть более жирный шрифт или небольшая иконка-папка у контейнеров.
- Для длинных подписей узлов: `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` с обязательным `title` с полным именем и тегом.
- Уровень вложенности: сохранить отступ 12px на уровень, при желании добавить тонкую вертикальную линию (border-left) по левому краю для группы детей, как в классических tree-view.

### 4.3 Панель свойств

- Группировка: блоки «Основные» (тип, имя, ID), при необходимости «Внешний вид» или «Данные», отдельно «События». Каждая группа — заголовок (малый, в цвете descriptionForeground) + отступ снизу.
- Тип элемента оформить как badge (некликабельный): отдельный блок с `border-radius` и фоном `var(--vscode-badge-background)` / цвет `var(--vscode-badge-foreground)`.
- Сделать раскладку полей гибкой: например, `label` с `min-width` и `input` с `flex: 1; max-width: 280px` или `width: 100%` внутри контейнера, чтобы на узкой панели поля сужались, а не обрезались.
- Визуально отделить заголовок выбранного элемента (имя + тег) от таблицы свойств: линия снизу или отступ; кнопки «Отмена» и «Сохранить» вынести в фиксированную полоску внизу панели свойств (sticky footer) или закрепить сверху блока свойств (sticky bar), чтобы они всегда были видны.

### 4.4 Превью

- Заменить «Превью не реализовано» на нейтральный empty state: краткая фраза («Визуальное превью формы пока не реализовано») и при необходимости подсказка, что структуру можно смотреть в дереве и в панели свойств.
- Оформить переключатель «Форма» / «Модуль» как вкладки в стиле VS Code: контейнер с двумя кнопками, активная — с подчёркиванием или отличимым фоном (`--vscode-tab-activeBackground` / border-bottom), неактивная — нейтральный фон/прозрачность, без стиля primary-кнопки.

### 4.5 Сплиттеры, отступы, пустые состояния

- Сплиттер: добавить визуальную «ручку» (несколько горизонтальных полос для вертикального сплиттера и вертикальных для горизонтального) по центру, сохраняя текущую логику перетаскивания; при hover подсвечивать полосу (например, `--vscode-focusBorder`).
- Привести отступы к одной шкале (4/8/12/16) во всех зонах и между элементами.
- Empty state: увеличить отступ (например 24px), при возможности добавить иконку (например, документ или форма) и короткий заголовок + подзаголовок; сохранять цвет `descriptionForeground` для подзаголовка.

### 4.6 Читаемость и контраст

- Заголовки зон (Элементы формы, Свойства, Превью): оставить размер около 0.9em, но цвет взять `var(--vscode-foreground)` с пониженной непрозрачностью (например 0.9) или оставить `descriptionForeground`, но убедиться, что в тёмной и светлой темах контраст достаточный.
- Плейсхолдер «Выберите элемент»: чуть больший отступ, при желании — простая иконка (например, стрелка или курсор) слева от текста.

### 4.7 Кнопки и статус сохранения

- Сохранить текущие переменные кнопок VS Code; при желании сделать кнопки «Отмена»/«Сохранить» одной строкой с иконками (например, отмена — крестик, сохранить — галочка), если в webview станут доступны codicon.
- Статус сохранения: компактная строка или иконка (галочка / спиннер) рядом с «Сохранить», без длинного текста при успехе; при ошибке — оставить текст ошибки или вынести в отдельную строку под кнопками с классом `.error`.

### 4.8 Доступность (рекомендации на будущее)

- Добавить для дерева роли `tree`, `treeitem`, атрибуты `aria-expanded`, `aria-selected`, навигацию с клавиатуры (стрелки, Enter для выбора).
- Для панели свойств — `role="region"` и `aria-label`.
- Стили `:focus-visible` для кнопок и узлов дерева (outline в цвет `--vscode-focusBorder`), чтобы фокус при навигации с клавиатуры был виден.

## 5. Приоритеты

| Приоритет | Направление | Предложения (кратко) |
|-----------|-------------|------------------------|
| Высокий   | Согласованность с VS Code | Фон дерева — sideBar; вкладки превью в стиле табов; единая шкала скруглений и отступов. |
| Высокий   | Панель свойств | Группировка свойств; sticky для кнопок Сохранить/Отмена; гибкая ширина полей. |

## 6. Ограничения (что не входит в план)

- Реализация полноценного визуального превью формы (рендер по макету) — отдельная функциональная задача.
- Изменение логики (сохранение, парсинг, drag-and-drop, открытие модуля) не затрагивается.
- `package.json` и команды расширения не меняются.
- В плане не описываются конкретные правки кода — только цели и предложения для последующей реализации.
