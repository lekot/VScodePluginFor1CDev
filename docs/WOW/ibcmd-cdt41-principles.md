# Принципы работы с ibcmd в CDT 41

> Норматив для реализации Infobase Manager, раскатки (deploy), матрицы e2e и gate подсистем.  
> См. также [ibcmd-api-reference.md](./ibcmd-api-reference.md) (сырой CLI) и исходники `src/services/ibcmd/`, `src/infobases/ibcmdConfigPathResolver.ts`.

**Связь с задачами:** YouTrack `1cviewer-12` (ibcmd / выгрузка–загрузка конфигурации).

---

## 1. Offline-режим и каталог `--data`

Утилита ibcmd в сценариях `infobase config import|export|check` для **автономного сервера** ожидает каталог данных автономного сервера **`--data=<пустой или рабочий каталог на сессию>`**. Без него на Windows при запуске через `spawn` без оболочки часто срабатывает «запасной» путь вида `%LocalAppData%\1C\1cv8\standalone-server\db-data\1Cv8.1CD.cfl` — **база не находится**, хотя в YAML указан корректный `infobase.file`.

**CDT 41:** на каждую операцию выделяется отдельный временный каталог под `--data`, после завершения — удаление (`ibcmdOfflineDataDir`).

---

## 2. Подключение к файловой ИБ: `--db-path` vs `--config`

Для **файловой** ИБ надёжная схема (как в vanessa-runner / **ibcmdrunner**):

- **`--db-path=<каталог ИБ>`** (каталог с данными файловой базы),
- плюс **`--data=…`**,
- учётные данные при необходимости **`--user` / `--password`**.

Только **`--config=<yaml>`** с `infobase.file` из процесса Node на части сборок **не привязывает** ИБ так же стабильно, как пара `--db-path` + `--data`.

**CDT 41:** если в записи каталога файловой ИБ или в явном YAML распознан `file:` на существующий каталог данных — готовим режим **`fileDb`** (`PreparedIbcmdFileDb`) и argv с `--db-path`. Серверная ИБ по-прежнему может идти через временный YAML + `--config`.

---

## 3. Порядок аргументов (import / export)

Сверка с ibcmdrunner:

| Операция  | Подключение | Далее | Позиционный аргумент в конце        |
|-----------|-------------|-------|-------------------------------------|
| `import`  | `--config` или `--db-path`, креды, `--data` | опц. `--extension` | **каталог выгрузки XML** (источник) |
| `export`  | то же       | опц. `--extension`, `--format`     | **каталог назначения**              |

**Не использовать** для export вариант `--out=` там, где сборка ibcmd ожидает **последний позиционный** каталог (как в ibcmdrunner).

---

## 4. Импорт: без `-F` / `--force` на ряде 8.3.27

На проверенных сборках **8.3.27** передача **`-F` или `--force`** для `infobase config import` даёт **ошибку разбора параметра**. Принудительная загрузка для import **не передаётся**; сценарий «перезапись конфигурации в ИБ» — это обычный import в уже доступную базу.

Для **`config check`** опциональный **`--force`** поддерживается отдельно (в т.ч. через `IBCMD_CONFIG_CHECK_FORCE` в gate).

---

## 5. Экспорт: каталог назначения должен быть пустым

Платформа при **`infobase config export`** в каталог выгрузки XML требует **пустой** каталог (иначе сообщение вида «каталог не пуст»).

**CDT 41:** если пользователь выбрал непустую папку и подтвердил продолжение, расширение **очищает содержимое** этой папки (файлы и подкаталоги), **не удаляя** саму выбранную директорию, затем запускает ibcmd. Пользователь должен явно согласиться с удалением содержимого.

---

## 6. Переменные окружения и явное подключение

Если в argv уже задано **`--config=`** или **`--db-path=`**, дочернему процессу **не передаются** `IBCMD_INFOBASE_CONFIG`, `IBCMD_USER`, `IBCMD_PASSWORD` из родительского env — иначе ibcmd может смешать «дефолтное» подключение с явным и давать ложные ошибки (блокировка другой базы, «база не обнаружена»).

---

## 7. Кодировка вывода в канал

Поток stdout/stderr от **spawn** у современного ibcmd часто уже в **UTF-8** (`[INFO] …`). Режим настройки **`auto`** для потоковой декодировки в CDT 41 трактуется как **UTF-8**. Если текст отображается неверно (legacy OEM), в настройках указать **`oem866`** или **`windows1251`**.

Одноразовый разбор буфера (`execFile`) в режиме `auto`: валидный UTF-8 → UTF-8, иначе на Windows — CP866.

---

## 8. Матрица и CLI-хелперы

Скрипты и матрица должны передавать **`--data`** так же, как продукт (`scripts/ibcmd-cli.cjs`, `test/matrix/ibcmdAdapter.ts`), чтобы поведение не расходилось с UI.

---

## 9. Версия ibcmd

Перед операциями с YAML-конфигом инфобазы выполняется проверка версии ibcmd; при слишком старой платформе показывается сообщение с требованием обновления (см. `ibcmdVersionSupport`).
