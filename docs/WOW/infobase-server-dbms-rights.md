# Серверная ИБ и права в СУБД (Infobase Manager)

> WOW §3B #52 — краткая памятка для разработчиков. Детали зависят от версии платформы 1С и выбранной СУБД; ориентируйтесь на официальную документацию «1С:Предприятие» по администрированию кластера и требованиям к СУБД.

## Зачем это нужно

Серверная информационная база хранит данные в СУБД (MS SQL Server, PostgreSQL и др.). Учётная запись СУБД, которую использует **рабочий процесс кластера 1С** (rphost / сервер приложений), должна иметь достаточные права на объекты базы данных, иначе возможны ошибки при создании ИБ, обновлении конфигурации, регламентных операциях и работе пользователей.

Расширение CDT 41 / Infobase Manager **не создаёт** серверную ИБ в кластере и **не настраивает** СУБД: оно добавляет уже существующую базу в локальный каталог и вызывает `ibcmd` / клиент 1С. Права в СУБД настраивает администратор.

## Типовые шаги администратора

1. **Создать пустую базу в СУБД** (или использовать шаблон, рекомендованный для вашей версии платформы).
2. **Выдать учётной записи сервера 1С** права, соответствующие роли «владелец БД» / полный доступ к схеме приложения — в точности как в документации 1С для вашей СУБД (набор ролей и объектов отличается между MS SQL и PostgreSQL).
3. **Зарегистрировать ИБ в кластере** (консоль администрирования, RAC и т.п.) и указать строку подключения к СУБД.
4. **Проверить запуск** толстого/тонкого клиента и при необходимости операции `ibcmd infobase config check` с YAML, сформированным для этой ИБ (см. реализацию в репозитории: `prepareIbcmdConfigYaml`).

## Связь со строкой `Srvr=…;Ref=…`

Строка вида `Srvr="cluster";Ref="Demo_UT";` описывает **точку входа в кластер 1С** (имя сервера или кластера и имя ИБ в списке кластера), а не логин СУБД. Учётные данные СУБД задаются на стороне сервера 1С при создании ИБ в кластере.

Пользователь и пароль **1С** (`Usr` / `Pwd` в строке подключения клиента) хранятся в каталоге расширения без пароля в `globalState`: пароль — в `SecretStorage` (см. `InfobaseEntry.hasStoredPassword` и `InfobaseStorageService`).
