# Манифест предсказуемости изменений (CDT 41)

Краткий контракт для контрибьюторов и ИИ-агентов: **что расширение обещает делать с файлами на диске**.

## 1. Источник истины

- Метаданные конфигурации — **файлы** в layout **Designer** или **EDT** внутри workspace.
- После операций CDT структура XML должна оставаться **загружаемой** платформой в типовых сценариях; сложные типы могут быть помечены как неподдерживаемые в матрице (`IBMATRIX_SKIP_TYPE_FOLDER_IDS`).

## 2. Операции записи

- CRUD элементов и правки свойств идут через **`XMLWriter`**, **`elementOperations`**, **`configurationXmlUpdater`** и специализированные сериализаторы (роли, формы).
- Внешние правки того же файла подхватываются **`MetadataWatcherService`** с перезагрузкой через **`ReloadCoordinatorService`**.

## 3. Границы

- CDT **не** выполняет полноценный статический анализ BSL и **не** заменяет конфигуратор/EDT для отладки и всех мастеров платформы.
- Редактор форм — догоняющий по зрелости; критерии в [edt-killer-target.md](analytics/edt-killer-target.md) уровень B.

## 4. Тестовые обязательства

Изменения в писателях XML и парсерах сопровождаются тестами в `test/suite/` и при необходимости кейсами матрицы / smoke (см. [README.md](../README.md)).
